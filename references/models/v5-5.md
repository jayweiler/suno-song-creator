# Suno v5.5

> **Released:** March 26, 2026 | **Status:** Current latest model
>
> Best-in-class prompt understanding, cue control, and feature set. Strong default
> for most songs. Read this file before writing the style prompt or lyrics when
> the user's target model is v5.5.

## Headline differences from v5

- **Voices** — clone the user's own voice and have Suno sing in it. Pro/Premier only,
  consent verification required. Distinct from the older Style Persona workflow.
- **Custom Models** — train a personalized model on the user's own catalog (up to 3,
  Pro/Premier). The model picks up the user's recurring style and voicing.
- **My Taste** — passive personalization layer. Suno learns from listens and likes
  in the background. No explicit picker UI. Influences defaults; an explicit Style
  prompt always overrides.
- **Replace Section (Pro)** — surgical regeneration of a single tagged section
  (`[Chorus]`, `[Bridge]`, etc.) without burning a full new generation.
- **Open in Studio** — post-generation fine-grained editing for mix tweaks
  (instrument volumes, reverb, panning).
- **Inspire (playlist-based)** — see "Inspire" section below.
- **Audio profile shift (important caveat):** v5.5 exports run roughly 1.5–2 dB
  quieter on integrated loudness with a meaningfully higher crest factor. The model
  delivers more headroom and more transient definition — a stem-friendly profile
  rather than a radio-finished one. **Regression for lo-fi, dark ambient, raw
  black metal, austere folk, or any aesthetic that requires the model to NOT
  polish.** For those cases, suggest v5 instead.

## Prompt format

**Narrative / NLP-driven.** v5.5 reads conversational descriptions of how the
music unfolds. Flowing sentences beat keyword lists, especially for nuanced
direction (arrangement timing, restraint, dynamic arc).

- ✅ "Begin with soft ambient layers, natural sounds, and a deep steady groove.
  Build gradually with flowing melodic synths and warm basslines. Final chorus
  stays at the dynamic of the verses — no triumphant lift."
- ❌ "ambient layers, deep groove, melodic synths, warm bass, no triumphant lift"

The narrative form lets the model time arrangement changes (when instruments
enter, when dynamics shift). Flat tag lists collapse into "all present from
start" and the dynamic directives get lost.

Tokenizer note: the first ~3 words of the style field carry more weight than
the last ten. Front-load priority — genre/subgenre first, then vocal direction,
then mood, then instruments, then BPM, then production notes.

## Style char limit & truncation

- **1000 characters.** Same as v4.5+ and v5.
- A well-crafted 400–600 char prompt usually outperforms a cluttered 900+ char one.
- Truncation strategy: front-load. If the prompt is going to lose its tail, the
  tail should be the least important content.

## Cue reliability — what to use

v5.5 reads parentheticals as an **internal control system** alongside section
tags. Cues carry real weight; treat them as deliberate composition, not garnish.

### Bracketed section-level performance modifiers (high reliability)

Pattern: `[<Performance Adjective> <Section Tag>]`. Apply to the whole tagged
section.

| Tag | Effect |
|---|---|
| `[Whispered Verse]` | Whole verse delivered as whisper |
| `[Spoken Word]` / `[Spoken Bridge]` / `[Spoken Verse]` | Entire section spoken, not sung |
| `[Belted Chorus]` | Power-vocal chorus |
| `[Haunting Whispered Pre-Chorus]` | Quiet, eerie texture |
| `[Staticky Spoken Pre-Chorus]` | Combination with timbral effect |
| `[Ethereal Female Whisper]` | Specific vocal character + delivery |

Combinations of one delivery word + one mood word + a section tag are the safe
pattern. Stack more and reliability drops.

### Inline parenthetical cues — vocal delivery (high reliability)

| Cue | Effect |
|---|---|
| `(spoken)` / `(spoken word)` | Spoken delivery for the line |
| `(whispered)` / `(whisper)` | Whispered delivery |
| `(half-sung)` | Between speaking and singing |
| `(belted)` | Powerful sung delivery |
| `(falsetto)` | High-register head voice |
| `(harmonized)` | Vocal harmonies on the line |
| `(ad-lib)` | Improvisational vocal |

### Inline cues — dynamics / arrangement-light (medium-to-high reliability)

| Cue | Effect |
|---|---|
| `(building intensity)` | Gradual lift in energy |
| `(stripped back)` / `(stripped)` | Pull instruments out |
| `(soft but defiant)` | Quiet volume + emotional weight |
| `(voice cracks then soars)` | Emotional break into power |
| `(half-time feel)` | Rhythmic feel shift |
| `(key change)` | Modulation |

### Inline cues — mood / character (medium reliability)

| Cue | Effect |
|---|---|
| `(haunting, intimate)` | Mood pair |
| `(playful)` | Lighter delivery |
| `(wry, half-smiling)` | Understated humor |
| `(quiet)` | Volume-down (pair with dynamic-controlling prompt) |

### Failure modes

- Suno can ignore cues if surrounding context strongly suggests otherwise (an
  "anthem" Style prompt overriding a `(whispered)` cue).
- First generation may not honor a cue; second often does. Re-roll once before
  assuming the cue is broken.
- A cue that lands in one generation may not land identically on regeneration.
  Suno is stochastic.

## v5.5-specific UI & features

### Remix/Edit submenu

Open a song → click Remix/Edit (circular arrow icon or menu item):

- **Open in Studio (New)** — full multitrack editing environment
- **Open in Editor (Pro)** — single-song editor
- **Cover** — recreate a song with different voice/style, preserving vocals/melody
- **Extend** — continue a song past its current end
- **Mashup** — blend two existing songs
- **Sample this song** — extract samples to reuse in new creations
- **Use as Inspiration (Pro)** — style transfer from the source song. Opens Custom
  Mode with the song loaded in the **Inspo** tab. This replaced the old "Create →
  Make Persona" workflow.
- **Adjust Speed**
- **Reuse Prompt** — pre-fills Custom Mode with the original generation parameters
- **Crop** — trim to specific regions
- **Replace Section (Pro)** — surgical regeneration of a single section while
  keeping the rest

### Create submenu

- **Voice (Pro)** — biometric voice upload (record or upload your own singing
  voice for use in generations). Requires consent checkbox. Beta. Pro/Premier only.
- **Remaster (Pro)**
- **Song Radio**

### Personas → Voices (critical rename)

The old "Personas" feature was renamed and split in v5.5:

- **"Voice"** now specifically = biometric voice upload (not style transfer).
- **"Style Persona from song" workflow → Remix/Edit → Use as Inspiration.**

These are commonly confused. When the user wants to use an existing song as a
style/vocal reference for a new song, point them at **Use as Inspiration**, not
Create → Voice.

### Inspo tab

Accessed via Use as Inspiration, or by clicking "Inspo" in the audio reference
tabs of Custom Mode. Accepts **up to 4 reference tracks** for blended style
transfer. Stack multiple songs to average voice/style across them.

When Inspo is loaded:
- Keep the Style prompt **shorter** — the reference track carries the baseline.
- Use the Style prompt only for what's *new* or needs explicit restraint
  directives.
- Suno's v5.5 guidance: 1–2 genres, 1 mood line, 2–4 priority instruments max.

### Inspire (playlist-based) — distinct from Inspo and My Taste

A separate playlist-picker variant of Inspo, surfaced via a dedicated UI: "Pick
a playlist of your own songs to inspire your next creation. Only your music will
be used to shape the sound."

- **Constraint:** **Own music only.** Cannot reference third-party songs or
  arbitrary public tracks. Any reference must be in one of the user's own
  playlists. (This is a v5.5 narrowing — earlier versions allowed broader Inspo
  sources via the Use as Inspiration → Inspo tab path.)
- **Mechanism:** Suno extracts musical characteristics from the chosen playlist
  and applies them to the new generation.
- **Best practice:** Curate a tight playlist of the user's own songs in the
  target register. A 20-song catch-all averages too broadly; a 3-song focused
  playlist gives a clearer signal.

Distinguish from:
- **My Taste** — passive background learning from listens/likes. No picker UI.
  Influences defaults; explicit Style prompt overrides.
- **Use as Inspiration / Inspo tab (older path)** — accepts up to 4 reference
  tracks; current v5.5 status of the broader-source variant is in flux. When in
  doubt, use the playlist-picker (Inspire) variant — its constraints are clear.

### Song Details (metadata editor)

Triple-dot menu → **Song Details** lets the user edit: Title, Caption, Short
Style tags (comma-separated genres/moods on song cards), Displayed Lyrics, Cover
Art, Visibility, Allow Comments, Allow Remix, Pin toggles.

**Song Details does NOT let you edit the long generation prompt** displayed
under the song title on the detail page. That text is preserved as generation
provenance and is permanent. Plan prompts accordingly — especially around
artist references, which become permanent on the published song page.

### Other v5.5 features

- **Get Stems / MIDI (Pro):** export separated stems for DAW post-processing.
- **Custom Models (Pro):** train a personalized model on the user's own catalog
  to capture style consistently.

## When to choose v5.5

- The user wants polished, professional-sounding output.
- The song is meant to use the user's own voice (Voices feature).
- The user has trained a Custom Model and wants to use it.
- The user wants surgical Replace Section editing.
- The user will mix the result in a DAW — v5.5's stem-friendly profile gives
  more headroom to work with.
- The song needs reliable, fine-grained cue control across many sections.

## When to choose v5 instead

- The song is meant to feel raw, lo-fi, austere, or unpolished. v5.5 over-polishes
  these aesthetics. v5 preserves more texture.
- The user wants the loudest, most "radio-finished" master.
- The user doesn't need any v5.5-only feature (Voices, Custom Models, Replace
  Section, Inspire) — v5 may simply give a sharper sound for the genre.
